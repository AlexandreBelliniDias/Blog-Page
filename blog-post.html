Prototypal inheritance

  • Alex
  • 28 January 2020
  • 45
  • 542

Special hidden property

In Javascipt we have a property called "Prototype" that can be set as null or reference another object.

If you want to use some propriety from the object but this propriety is missing. JavaScript retrieves this information from the Prototype Object that you linked before.

To set up your link between objects we need to use the "getter/setter" called "__proto__".

An object can only inherit from one Prototype and the reference can not generate a circle.

In this example, we defined "animal" with a propriety "eats" as true, "rabbit" with propriety "jumps" also as true and we linked animals as rabbit prototypes.

When we called the missing propriety "eats" for rabbit, JavaScript automatically takes the variable from animal and return as true.

This inheritance can also be used for methods and can be extended.

In the example, we defined animals with attack and sleep method, the carnivore with the propriety eat meat and the wolf as empty.

To be able to call wof.attack and wolf.sleep methods and wolf.eat propriety we linked "wolf" to "carnivore" and "carnivore" to "animal" in an extended chain.

A "for loop" can be used to call all the object proprieties and we can use an "if" statement to differentiate what is its own proprieties and what is an inheritance from other objects.

When you call some object propriety, it will return "this" propriety even when the prototype object has the same object propriety.
